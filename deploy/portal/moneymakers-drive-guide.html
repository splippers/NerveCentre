Google Drive uploads (projectscan)

Upload portfolio reports from the moneymakers dashboard at http://192.168.1.2/projects/ or from the CLI. This guide covers Google Workspace, OAuth, systemd, SSH, and optional email lockdown.

1. What you install

On the host that runs moneymakers/projectscan.py:

python3 -m venv .venv
.venv/bin/pip install -r requirements-google.txt

Depends are listed in moneymakers/requirements-google.txt.

2. Google Cloud (Workspace internal app)

Open the same Google Cloud project everywhere (Console works in a desktop or mobile browser).

  1. Create / pick a GCP project.
  2. Enable APIs & Services → Library → enable “Google Drive API”.
  3. OAuth consent screen: user type Internal (Workspace — only organisational accounts).
  4. Add scopes (manually if the UI asks):
    https://www.googleapis.com/auth/drive.file
    https://www.googleapis.com/auth/userinfo.email
  5. Credentials → Create → OAuth client ID → Desktop app. Download the JSON.

Save it as moneymakers/project_index/client_secrets.json (parent dir is fine; it is created when you run a scan).

If users cannot consent, Workspace admin API controls might block Drive: Google Admin Console → Security → Access and data control → API controls.

3. Secrets, token paths, environment

Variable Role
PROJECTSCAN_PUBLIC_ORIGINLAN portal base for dashboard links — set to http://192.168.1.2.
PROJECTSCAN_DRIVE_CLIENT_SECRETSPath to Desktop OAuth JSON (defaults to project index).
PROJECTSCAN_DRIVE_TOKEN_PATHWhere OAuth refresh token lives (defaults next to index).
PROJECTSCAN_DRIVE_FOLDER_IDOptional Drive folder ID for uploads.
PROJECTSCAN_DRIVE_ALLOWED_EMAILSComma-separated lowercased logins allowed to upload (optional).
PROJECTSCAN_DRIVE_OAUTH_PORTFixed loopback port for OAuth when using SSH (see below).

4. Download-only (upload to Drive in the browser)

No OAuth required: on the prioritiser toolbar choose Report format and Report subset, then Download report.

Direct GET (bookmark or script):

http://192.168.1.2/projects/api/report/download?format=txt&subset=all

Replace format with md, csv, or json; subset with past_metaai if needed.

5. First sign-in: drive-auth

From moneymakers/ (venv Python preferred):

export PROJECTSCAN_DRIVE_ALLOWED_EMAILS="you@yourdomain.com" # optional
.venv/bin/python projectscan.py drive-auth

A browser opens; sign in with the Workspace account. The token file is written (default near project_index/).

6. OAuth over SSH

Desktop OAuth redirects to http://127.0.0.1:<port>/.... If projectscan.py drive-auth runs on a remote server, forward that loopback port from your laptop before you start auth:

# On your laptop (second terminal — leave running):
ssh -L 9876:127.0.0.1:9876 -N YOUR_USER@REMOTE_HOST

Then on REMOTE_HOST:

export PROJECTSCAN_DRIVE_OAUTH_PORT=9876
python3 projectscan.py drive-auth --oauth-port 9876

Open the authorisation URL in a browser on the laptop; the redirect goes to local :9876, tunnels to the server, and completes OAuth.

Multi-hop SSH: attach -L to the hop that terminates on the machine running Python.

7. Lock uploads to specific emails

export PROJECTSCAN_DRIVE_ALLOWED_EMAILS="jon@splippers.com,colleague@company.com"

Combined with an Internal OAuth app, Workspace already limits who can consent; this variable is app-side defence on your server.

8. Production (systemd + nginx)

You browse the dashboard at http://192.168.1.2/projects/ via nginx reverse proxy. Persist Drive-related env here:

PROJECTSCAN_PUBLIC_ORIGIN=http://192.168.1.2
PROJECTSCAN_DRIVE_CLIENT_SECRETS=/path/to/client_secrets.json
PROJECTSCAN_DRIVE_ALLOWED_EMAILS=jon@splippers.com
# PROJECTSCAN_DRIVE_OAUTH_PORT=9876 # only for SSH OAuth

After edits: sudo systemctl restart projectscan-dashboard.

9. Interactive wizard (URLs & copy-paste commands)

From the checkout:

./drive_workspace_setup_wizard.py --project-id YOUR_GCP_PROJECT --allow-email jon@splippers.com

Runs through Console links (mobile-friendly), env snippets, and optional drive-auth. Script path: moneymakers/drive_workspace_setup_wizard.py.
